# Как опубликовать проект AI Lawyer в GitHub

Пошаговая инструкция для тех, кто недавно зарегистрировался в GitHub.

---

## Шаг 1. Установить Git (если ещё не установлен)

- **Mac:** откройте Терминал и введите: `git --version`. Если команда не найдена, установите Xcode Command Line Tools: `xcode-select --install`.
- Или скачайте Git с [git-scm.com](https://git-scm.com/).

Проверка:
```bash
git --version
```

---

## Шаг 2. Настроить имя и email в Git (один раз на компьютере)

В Терминале выполните (подставьте свои данные):

```bash
git config --global user.name "Ваше Имя"
git config --global user.email "ваш_email@example.com"
```

Email лучше указать тот, с которым вы зарегистрированы на GitHub.

---

## Шаг 3. Создать репозиторий на GitHub

1. Зайдите на [github.com](https://github.com) и войдите в аккаунт.
2. Нажмите зелёную кнопку **New** (или **+** в правом верхнем углу → **New repository**).
3. Заполните:
   - **Repository name:** например `ai-lawyer` или `AI-Lawyer`.
   - **Description:** по желанию, например «AI Lawyer — мобильное приложение для анализа документов».
   - **Public** — выберите Public, чтобы коллеги могли смотреть и клонировать проект.
   - **НЕ ставьте** галочку «Add a README file» — проект уже есть у вас локально.
4. Нажмите **Create repository**.

После создания GitHub покажет страницу с подсказками. **URL репозитория** будет таким:
`https://github.com/ВАШ_ЛОГИН/ai-lawyer.git`  
(или с именем, которое вы ввели). Этот адрес понадобится в шаге 5.

---

## Шаг 4. Закоммитить все файлы проекта (уже сделано из инструкции)

В папке проекта в Терминале:

```bash
cd "/Users/amirkhalimov/Cursor/AI Lawyer"
git add .
git status
git commit -m "Initial commit: AI Lawyer app"
```

Так вы добавляете все файлы в Git и делаете первый коммит. Файлы из `.gitignore` (например `node_modules`, `.env`) в репозиторий не попадут.

---

## Шаг 5. Связать локальный проект с GitHub и отправить код

Замените `ВАШ_ЛОГИН` и `ai-lawyer` на ваш логин и имя репозитория:

```bash
git remote add origin https://github.com/ВАШ_ЛОГИН/ai-lawyer.git
git branch -M main
git push -u origin main
```

При первом `git push` браузер или терминал могут запросить вход в GitHub (логин и пароль или токен). Если просят пароль — в GitHub теперь используют **Personal Access Token** вместо пароля аккаунта.

---

## Как коллеги могут смотреть и изменять проект

### Посмотреть код
- Откройте в браузере: `https://github.com/ВАШ_ЛОГИН/ai-lawyer`.
- Все файлы и история коммитов будут видны там.

### Скачать проект к себе (клонировать)
```bash
git clone https://github.com/ВАШ_ЛОГИН/ai-lawyer.git
cd ai-lawyer
npm install
```

### Дать право вносить изменения
1. На странице репозитория: **Settings** → **Collaborators** → **Add people**.
2. Введите GitHub-логин или email коллеги и выдайте доступ (Collaborator).
3. Коллега клонирует репозиторий, вносит изменения, затем:
   ```bash
   git add .
   git commit -m "Описание изменений"
   git push
   ```

---

## Полезные команды на будущее

| Действие | Команды |
|--------|--------|
| Посмотреть статус | `git status` |
| Добавить изменения и закоммитить | `git add .` → `git commit -m "описание"` |
| Отправить на GitHub | `git push` |
| Забрать изменения с GitHub | `git pull` |

---

## Важно: секреты и ключи

- Файл **`.env`** в репозиторий не попадает (он в `.gitignore`).
- Для коллег создайте файл **`.env.example`** с примером переменных (без реальных ключей) и добавьте в репозиторий, чтобы они знали, что нужно настроить у себя.

Если что-то пойдёт не так на каком-то шаге — напишите, на каком именно, и какой текст ошибки видите.
